# Настройка Supabase для проекта Facade Orders

## Шаг 1: Создание проекта в Supabase

1. Перейдите на [supabase.com](https://supabase.com)
2. Нажмите "Start your project"
3. Войдите через GitHub (рекомендуется)
4. Создайте новый проект:
   - **Name**: `facade-orders-db`
   - **Database Password**: создайте надежный пароль (сохраните его!)
   - **Region**: выберите ближайший к вам регион
   - **Pricing Plan**: Free

## Шаг 2: Получение данных подключения

После создания проекта:

1. Перейдите в **Settings** → **Database**
2. Найдите секцию **Connection string**
3. Скопируйте **URI** (он выглядит как `postgresql://postgres:[password]@db.[project-ref].supabase.co:5432/postgres`)

## Шаг 3: Настройка переменных окружения

Создайте файл `.env` в корне проекта:

```env
# Supabase Database
DATABASE_URL=postgresql://postgres:[YOUR_PASSWORD]@db.[YOUR_PROJECT_REF].supabase.co:5432/postgres

# Другие переменные
SECRET_KEY=your-super-secret-key-here
FLASK_ENV=production
```

## Шаг 4: Установка дополнительных зависимостей

Убедитесь, что в `requirements.txt` есть:

```
psycopg2-binary==2.9.7
python-dotenv==1.0.0
```

## Шаг 5: Запуск миграции

Выполните команды:

```bash
# Установка зависимостей
pip install -r requirements.txt

# Инициализация базы данных
python supabase_init.py
```

## Шаг 6: Проверка подключения

Запустите тест подключения:

```bash
python test_supabase_connection.py
```

## Преимущества Supabase

- ✅ **2 ГБ бесплатного хранилища**
- ✅ **Полноценный PostgreSQL**
- ✅ **Веб-интерфейс для управления данными**
- ✅ **Автоматические бэкапы**
- ✅ **API для прямого доступа к данным**
- ✅ **Реальное время (Real-time subscriptions)**
- ✅ **Аутентификация и авторизация**

## Лимиты бесплатного плана

- 2 ГБ базы данных
- 500 МБ файлового хранилища
- 2 ГБ пропускной способности
- 50,000 запросов в месяц

## Мониторинг использования

В панели Supabase вы можете отслеживать:
- Использование базы данных
- Количество запросов
- Размер хранилища
- Активность API

## Бэкапы

Supabase автоматически создает бэкапы:
- Ежедневные бэкапы (хранятся 7 дней)
- Еженедельные бэкапы (хранятся 4 недели)

## Безопасность

- Все соединения зашифрованы (SSL/TLS)
- Row Level Security (RLS) для контроля доступа
- API ключи для безопасного доступа
- Возможность настройки IP-фильтрации
