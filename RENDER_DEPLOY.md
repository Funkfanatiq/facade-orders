# Развертывание на Render.com

## Пошаговая инструкция

### 1. Подготовка репозитория

1. **Создайте репозиторий на GitHub** (если еще не создан)
2. **Загрузите код** в репозиторий
3. **Убедитесь, что все файлы загружены:**
   - `app.py`
   - `models.py`
   - `config.py`
   - `requirements.txt`
   - `Procfile`
   - `init_db.py`
   - `templates/` (папка с шаблонами)
   - `static/` (папка со статическими файлами, если есть)

### 2. Создание базы данных PostgreSQL

1. **Войдите в Render.com**
2. **Создайте новую PostgreSQL базу данных:**
   - Нажмите "New +" → "PostgreSQL"
   - Выберите план (Free для тестирования)
   - Назовите базу данных (например: `facade-orders-db`)
   - Выберите регион
   - Нажмите "Create Database"

### 3. Создание веб-сервиса

1. **Создайте новый Web Service:**
   - Нажмите "New +" → "Web Service"
   - Подключите ваш GitHub репозиторий
   - Выберите репозиторий с кодом

2. **Настройте сервис:**
   - **Name:** `facade-orders-app`
   - **Environment:** `Python 3`
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `gunicorn app:app`

### 4. Настройка переменных окружения

В настройках веб-сервиса добавьте переменные окружения:

```
SECRET_KEY=your-super-secret-key-here-change-this
DATABASE_URL=postgresql://username:password@host:port/database
RENDER=true
```

**Важно:** 
- `SECRET_KEY` - замените на случайную строку
- `DATABASE_URL` - скопируйте из настроек PostgreSQL базы данных
- `RENDER=true` - указывает, что приложение работает в продакшене

### 5. Инициализация базы данных

После первого деплоя выполните инициализацию базы данных:

1. **В настройках веб-сервиса** добавьте команду инициализации:
   - **Build Command:** `pip install -r requirements.txt && python init_db.py`
   - Или выполните инициализацию вручную через Render Shell

### 6. Настройка домена (опционально)

1. **В настройках веб-сервиса** можно настроить кастомный домен
2. **Или используйте** предоставленный Render.com домен

## Доступ к приложению

После успешного деплоя приложение будет доступно по адресу:
`https://your-app-name.onrender.com`

### Тестовые аккаунты

После инициализации базы данных будут созданы следующие аккаунты:

- **Администратор:** `admin` / `admin123`
- **Менеджер:** `manager` / `manager123`
- **Фрезеровка:** `frez` / `frez123`
- **Шлифовка:** `shlif` / `shlif123`
- **Производство:** `prod` / `prod123`
- **Монитор:** `monitor` / `monitor123`

## Особенности продакшена

1. **Облачное хранение отключено** - в продакшене используется только PostgreSQL
2. **Статические файлы** обслуживаются Render.com
3. **Логи** доступны в панели управления Render
4. **Автоматические деплои** при обновлении кода в GitHub

## Мониторинг и логи

- **Логи приложения** доступны в панели Render
- **Метрики производительности** в разделе Metrics
- **Статус сервиса** в разделе Health

## Обновление приложения

1. **Внесите изменения** в код
2. **Загрузите изменения** в GitHub
3. **Render автоматически** пересоберет и перезапустит приложение

## Устранение неполадок

### Проблемы с базой данных
- Проверьте переменную `DATABASE_URL`
- Убедитесь, что база данных PostgreSQL создана
- Выполните инициализацию: `python init_db.py`

### Проблемы с зависимостями
- Проверьте `requirements.txt`
- Убедитесь, что все зависимости указаны

### Проблемы с запуском
- Проверьте `Procfile`
- Убедитесь, что команда запуска корректна
- Проверьте логи в панели Render

## Безопасность

1. **Измените пароли** по умолчанию после первого входа
2. **Используйте сильный SECRET_KEY**
3. **Настройте HTTPS** (включен по умолчанию на Render)
4. **Регулярно обновляйте** зависимости

## Резервное копирование

- **PostgreSQL база данных** автоматически резервируется Render
- **Код** хранится в GitHub
- **Настройки** в переменных окружения Render
