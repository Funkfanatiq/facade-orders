## 🔧 Настройка переменных окружения:

### Для локальной разработки:
Создайте файл `.env` в корне проекта (или добавьте строки в существующий):
```env
MAIL_USERNAME=ваш_почта@mail.ru
MAIL_PASSWORD=пароль_приложения_из_настроек
```

### Для Render.com:
**Важно:** файл `.env` не попадает на сервер. Добавьте переменные вручную:

1. Откройте ваш сервис на [Render Dashboard](https://dashboard.render.com)
2. Settings → Environment → Environment Variables
3. Добавьте:
   - **Key:** `MAIL_USERNAME` **Value:** `pictorspb@mail.ru` (или ваш адрес)
   - **Key:** `MAIL_PASSWORD` **Value:** пароль приложения Mail.ru (из настроек «Безопасность»)
4. Сохраните — Render пересоберёт приложение

## 🔧 Render.com — ограничения:

**На бесплатном тарифе** Render блокирует исходящие подключения к SMTP (порты 25, 465, 587) с сентября 2025. IMAP (порт 993) может работать или блокироваться.

- **Отправка писем (SMTP)** — требует **платный план** Render (Starter $7/мес и выше)
- **Получение (IMAP)** — попробуйте кнопку **«🔌 Проверить»** во вкладке Почта, чтобы увидеть точную ошибку
- **Если ошибка `Connection refused` или таймаут** — это блокировка Render. Решение: перейти на платный план или хостинг с открытыми портами (Railway, Fly.io, VPS)

## 🔐 Настройка пароля приложения Mail.ru:

1. Зайдите в настройки почты Mail.ru
2. Перейдите в "Безопасность"
3. Включите "Пароли приложений"
4. Создайте новый пароль для приложения
5. Используйте этот пароль в настройках

## 📥 IMAP — получение писем:

Почтовый агент загружает письма с Mail.ru по IMAP (сервер `imap.mail.ru:993`). Используются те же переменные окружения:
- `MAIL_USERNAME` — адрес ящика
- `MAIL_PASSWORD` — пароль приложения Mail.ru

### Как использовать:
1. Нажмите **«📥 Получить письма»** во вкладке «Почта».
2. Приложение подключится к IMAP и загрузит до 200 последних писем (новые — только те, которых ещё нет в БД).
3. Автоматическая проверка новых писем каждые 60 секунд.

### Папки (как в Mail.ru):
- Входящие, Отправленные, Черновики, Архив, Спам, Корзина
